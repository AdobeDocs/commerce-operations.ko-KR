# Adobe Commerce 2.4.3 릴리스 노트

Adobe Commerce 2.4.3에서는 성능 및 보안 개선 사항과 주목할 만한 플랫폼 개선 사항이 도입되었습니다. 향상된 보안 기능에는 reCAPTCHA 적용 범위 확장 및 기본 제공 비율 제한 포함 등이 있습니다. 코어 작성기 종속성 및 타사 라이브러리가 PHP 8.x와 호환되는 최신 버전으로 업그레이드되었습니다.

이 릴리스에는 핵심 코드에 대한 370개가 넘는 새로운 수정 사항과 33개의 보안 개선 사항이 포함되어 있습니다. Adobe Commerce 2.4.2 릴리스 노트에서 식별된 모든 알려진 문제는 이 릴리스에서 해결되었습니다.

{{bics}}

자세한 내용은 [Adobe Commerce 2.4.2-p2 릴리스 노트](../security/2-4-2-p2.md) Adobe Commerce 2.4.2-p2에 대한 정보입니다.

## 기타 릴리스 정보

이러한 기능에 대한 코드가 분기별 릴리스와 함께 번들로 제공되지만, 이러한 프로젝트(예: B2B, Page Builder 및 Progressive Web Application(PWA) Studio)의 몇 가지 도 독립적으로 릴리스됩니다. 이러한 프로젝트에 대한 버그 수정 사항은 각 프로젝트의 설명서에서 사용할 수 있는 프로젝트별 릴리스 정보에 설명되어 있습니다.

## API 속도 제한이 있는 문제를 해결하기 위해 MC-43048__set_rate_limits__2.4.3.patch를 적용합니다

이 핫픽스는 웹 API가 배열에 20개 이상의 항목이 포함된 요청을 처리할 수 없는 문제에 대한 솔루션을 제공합니다. 이 문제는 Magento Open Source 2.4.3, Adobe Commerce 2.4.3 또는 2.3.7-p1을 실행하는 배포에 영향을 줍니다. DoS(Denial-of-Service) 공격을 방지하기 위해 기본 제공 속도 제한이 이러한 릴리스에 추가되었으며, 기본 최대값이 20으로 설정되었습니다. 이 패치는 기본 제한을 더 높은 값으로 되돌립니다. 스토어에 DoS 공격이 있다고 의심되는 경우 Adobe은 기본 입력 제한을 낮은 값으로 낮추어 요청할 수 있는 리소스 수를 제한하는 것을 권장합니다. 자세한 내용은 [웹 API가 배열에 20개 이상의 항목이 있는 요청을 처리할 수 없습니다](https://support.magento.com/hc/en-us/articles/4406893342093) 기술 자료 문서입니다.

## AC-384__Fix_Uncompatible_PHP_Method__2.4.3_ce.patch를 적용하여 업그레이드 시 PHP 치명적인 오류를 해결합니다

Adobe Commerce 2.4.3으로 업그레이드하는 동안 다음 치명적인 오류가 발생할 수 있습니다.

```terminal
PHP Fatal error: Uncaught Error: Call to undefined function Magento\Framework\Filesystem\Directory\str_contains() in [...]/magento/vendor/magento/framework/Filesystem/Directory/DenyListPathValidator.php:74
```

이 오류는 `str_contains` PHP 8.x 함수인 함수 Adobe Commerce 2.4.3은 PHP 8.x를 지원하지 않습니다. 이 핫픽스는 이 함수를 지원되는 PHP 7.x 함수로 바꿉니다. 자세한 내용은 [Adobe Commerce 업그레이드 2.4.3, 2.3.7-p1 PHP 치명적 오류 핫픽스](https://support.magento.com/hc/en-us/articles/4408021533069-Adobe-Commerce-upgrade-2-4-3-2-3-7-p1-PHP-Fatal-error-Hotfix) 기술 자료 문서입니다.

## 적용 `AC-3022.patch` DHL을 계속 운송 운송업체로 제공

DHL에서 스키마 버전 6.2를 도입했으며 조만간 스키마 버전 6.0을 사용하지 않을 예정입니다. DHL 통합을 지원하는 Adobe Commerce 2.4.4 및 이전 버전은 버전 6.0만 지원합니다. 이러한 릴리스를 배포하는 판매업체는 이 버전을 적용해야 합니다 `AC-3022.patch` 그들이 편의상 DHL을 운송 회사로 계속 제공할 수 있도록. 자세한 내용은 [DHL을 배송 반송파로 계속 제공하기 위해 패치를 적용합니다.](https://support.magento.com/hc/en-us/articles/7707818131597-Apply-a-patch-to-continue-offering-DHL-as-shipping-carrier) 패치 다운로드 및 설치에 대한 자세한 내용은 기술 자료 문서를 참조하십시오.

## 밝은 영역

### 대폭 향상된 보안 기능

이 릴리스에는 33개의 보안 수정 사항과 플랫폼 보안 개선 사항이 포함되어 있습니다. 이러한 보안 수정 사항 중 대부분은 2.4.2-p2 및 2.3.7-p1로 지원됩니다.

#### RCE(원격 코드 실행) 및 XSS(크로스 사이트 스크립팅) 취약점을 해결하는 33가지 보안 개선 사항

지금까지 이 문제와 관련된 확인된 공격은 발생하지 않았습니다. 그러나 특정 취약점은 고객 정보에 액세스하거나 관리자 세션을 인계하기 위해 악용될 수 있습니다. 이러한 문제 대부분은 공격자가 먼저 관리자에 액세스할 수 있어야 합니다. 그 결과, 다음 작업을 포함하되 이에 제한되지 않는 등 관리자를 보호하기 위해 필요한 모든 조치를 취하도록 요구합니다. IP 허용 목록에 추가, [이중 인증](https://devdocs.magento.com/guides/v2.4/security/two-factor-authentication.html), VPN 사용, 고유한 위치 사용 `/admin`그리고 암호 위생도 좋습니다 자세한 내용은 [Adobe 보안 공지](https://helpx.adobe.com/security/products/magento/apsb21-64.html) 이 해결된 문제에 대해 토론해 보십시오.

#### 추가적인 보안 개선 사항

이 릴리스의 보안 개선 사항은 다음을 포함한 최신 보안 모범 사례를 준수하도록 개선합니다.

* A **새 작성기 플러그인** 종속성 혼동을 방지하고 공개 패키지 리포지토리의 내부 패키지와 동일한 이름으로 악성 패키지를 식별하는 데 도움이 됩니다. 자세한 내용은 [Adobe 릴리스 새 작성기 플러그인(2.4.3 릴리스 포함)](https://magento.com/blog/best-practices/adobe-releases-new-composer-plugin-magento-243-release) 블로그 게시물.

* **이제 비율 제한이 API에 내장되어 있습니다** 서비스 거부(DoS) 공격을 방지합니다. 이제 웹 API는 클라이언트가 요청할 수 있는 리소스의 크기 또는 수에 제한을 적용합니다(기본 최대값은 20으로 설정되고 비즈니스 요구 사항에 따라 다른 값으로 구성할 수 있음). 자세한 내용은 [비율 제한](https://developer.adobe.com/commerce/webapi/get-started/api-security/) 를 참조하십시오. <!-- MC-35358-->

* **ReCAPTCHA 범위가 확장되었습니다** 다음을 포함합니다.

   * 해당 HTML 페이지가 있는 웹 API는 ReCAPTCHA를 통해 다룹니다. (통합으로 액세스하는 웹 API는 제외합니다.) ReCAPTCHA 커버리지는 스팸 공격으로부터 끝점을 보호합니다. OAuth를 사용하는 타사 통합 서비스에서 웹 API에 액세스하면 ReCAPTCHA가 비활성화됩니다.<!-- MC-34472-->

   * 주문 상점 전면 페이지 및 결제 관련 웹 API. 이러한 페이지에 대한 ReCAPTCHA 보호는 기본적으로 비활성화되어 있으며 관리자로부터 활성화할 수 있습니다. 이 보도는 상점들을 폭행하는 것으로부터 보호하기 위한 반폭력 메커니즘을 더합니다.

**새로 만들기 `use_application_lock` 색인 모드**. 다음 `use_application_lock` 모드를 사용하면 환경 변수를 사용하거나 `app/etc/env.php` 파일. 이 모드를 활성화한 상태에서 실패한 후 더 이상 색인을 수동으로 재설정할 필요가 없습니다. 자세한 내용은 [재인덱스 프로세스에 응용 프로그램 잠금 모드 사용](https://developer.adobe.com/commerce/php/development/components/indexing/#using-application-lock-mode-for-reindex-processes).

### 플랫폼 개선 사항

버전 2.4.3은 아직 PHP 8.x와 호환되지 않지만, 다음 플랫폼 업그레이드를 통해 PHP 8.x와의 향후 호환성을 더욱 강화할 수 있습니다.

* 코어 작성기 종속성 및 타사 라이브러리가 PHP 8.x와 호환되는 최신 버전으로 업그레이드되었습니다. <!-- MC-39514-->

* CkoutJS 라이브러리가 v3.5.1(최신 버전)으로 업그레이드되었습니다. <!-- MC-40694-->

* 더 이상 사용되지 않는 SmallMCE v3 라이브러리가 제거되었습니다. 다음 `Magento_Tinymce3Banner` Adobe Commerce에서 SmallMCE v3.x와 관련된 모듈 및 MFTF 테스트를 제거했습니다. <!-- MC-42199 42170 -->

* 버전 2.4.3은 Redis 6.0.12와 호환되는지 테스트되고 확인되었습니다. (버전 2.4.x는 Redis 5.x와 호환됩니다.)

* Laminas 라이브러리 종속성이 PHP 8.x 호환 버전으로 업그레이드되었습니다. 일부 중복 종속성이 `composer.json` 파일. **Adobe Commerce 2.4.3 에서는 Laminas 3.4.0 사용**. <!-- MC-39513-->

## 알려진 문제

**문제**: _reCAPTCHA가 비활성화된 경우 JavaScript 오류_. reCAPTCHA가 체크아웃에 비활성화되어 있으면 체크아웃은 진행되지만, 애플리케이션에서 다음을 표시합니다 `Uncaught TypeError` 오류가 콘솔 로그에 표시됩니다.  이 문제는 이후 릴리스에서 수정됩니다. <!-- MC-42589 42649-->

**문제**: _컨텐츠 보안 정책 오류_. storfront는 콘솔 로그에 다음 오류가 표시됩니다. `The Content-Security-Policy directive frame-ancestors does not support the source expression unsafe-inline`. 저장소 성능이 영향을 받지 않습니다. <!-- MC-42613-->

**문제**: _선물 카드 금액의 PayPal 신용 표시가 있는 예외 항목_. PayPal Credit가 사용으로 설정되고 복수 기프트 카드 금액이 구성된 경우, 쇼핑객이 기프트 카드 값에 대한 금액을 변경하면, 상점 전체는 할부 지급 금액을 갱신하지 않습니다. 이 문제에 대한 수정 사항이 Adobe Commerce 2.4.4 및 Magento Open Source 2.4.4에 포함되어 있습니다.<!-- MC-42499 AC-344-->

**문제**:  _다음&#x200B;**주문에 추가**SKU별로 장바구니에 추가된 제품에 대해 단추가 작동하지 않습니다_.  응용 프로그램에서 **주문에 추가** SKU별로 주문에 제품을 추가한 후 단추: `An error has happened during application run`. 제품이 장바구니에 추가되지 않습니다. **해결 방법**: 제품 추가 기능을 사용합니다. <!-- AC-345-->

**문제**: GraphQL `category` 및 `categoryList` 쿼리는 공유 카탈로그의 카테고리를 숨길지 또는 노출되는지를 결정하는 카테고리 권한을 무시합니다. 현재 Storfront는 공유 카탈로그에 할당되거나 할당되지 않은 모든 범주를 표시합니다. 이 도구는 B2B 공유 카탈로그가 활성화된 PWA 저장소를 구현하는 Adobe Commerce 2.4.3 배포에 영향을 줍니다. <!-- MC-42528-->

## 마이그레이션 도구 키트

데이터 마이그레이션 도구는 기존 Magento 1.x 저장소 데이터를 Magento 2.x로 전송하는 데 도움이 됩니다. 이 명령줄 인터페이스에는 확인, 진행 추적, 로깅 및 테스트 기능이 포함됩니다. 설치 지침은 [데이터 마이그레이션 도구 설치](../../../tools/data-migration-tool/install.md). 탐색 또는 참가 고려 [데이터 마이그레이션 저장소](https://github.com/magento/data-migration-tool).

다음 [코드 마이그레이션 툴킷](https://github.com/magento-commerce/code-migration) 기존 Magento 1.x 저장소 확장 및 사용자 지정을 Magento 2.x에 전송하는 데 도움이 됩니다. 명령줄 인터페이스는 Magento 1.x 모듈 및 레이아웃을 변환하는 스크립트를 포함합니다.
