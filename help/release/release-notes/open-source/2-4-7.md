# Magento Open Source 2.4.7-beta1 릴리스 노트

>[!IMPORTANT]
>
>베타 릴리스에는 결함이 포함될 수 있으며 어떠한 종류의 보증도 없이 &quot;있는 그대로&quot; 제공됩니다. Adobe은 베타 릴리스를 유지, 수정, 업데이트, 변경, 수정 또는 지원(Adobe 지원 서비스 등을 통해)할 의무가 없습니다. 고객은 베타 릴리스 및/또는 관련 설명서나 자료의 올바른 기능이나 성능에 어떠한 의존도 하지 말고 주의하는 것이 좋습니다. 따라서 Beta 릴리스를 사용하는 것은 전적으로 고객 자신의 책임입니다.

Magento Open Source 2.4.7-beta1에는 140개 이상의 품질 수정 사항 및 개선 사항이 포함되어 있습니다. Core Composer 종속성 및 타사 라이브러리가 사용 가능한 최신 버전으로 업그레이드되었습니다. Composer 2.5에 대한 지원이 추가되었지만 Composer 2.2.x는 계속 지원됩니다.

이 릴리스에서는 사용자 지정 특성 및 향상된 GraphQL 캐싱 기능에 대한 GraphQL 적용 범위를 강화합니다.

## 기타 릴리스 정보

이러한 기능에 대한 코드가 Magento Open Source 코어 코드 릴리스와 함께 번들로 제공되지만, 이러한 프로젝트 중 일부도 독립적으로 릴리스됩니다. 이러한 프로젝트에 대한 버그 수정은 각 프로젝트에 대한 설명서에서 사용할 수 있는 별도의 프로젝트별 릴리스 정보에 설명되어 있습니다.

## Magento Open Source 2.4.7-베타1

Magento Open Source 2.4.7-beta1은 다음 구성 요소 버전에 대해 테스트되었습니다.

* 작성기 2.5
* Elasticsearch 8.5
* MariaDB 10.6
* MySQL 8.0
* OpenSearch 2.5
* PHP 8.2 및 8.1
* RabbitMQ 3.11
* Redis 7.0
* 바니시 7.3

우리의 기술 스택은 PHP와 MySQL을 기반으로 구축됩니다. 다음을 참조하십시오 [시스템 요구 사항](https://experienceleague.adobe.com/docs/commerce-operations/installation-guide/system-requirements.html).

### 향상된 보안 기능

이 릴리스에는 13가지 보안 수정 사항 및 플랫폼 보안 개선 사항이 포함되어 있습니다.

현재까지 이러한 문제와 관련된 확인된 공격은 발생하지 않았습니다. 그러나 특정 취약성은 고객 정보에 액세스하거나 관리자 세션을 인수하는 데 잠재적으로 악용될 수 있습니다. 이러한 문제의 대부분은 공격자가 먼저 관리자에 대한 액세스 권한을 얻어야 합니다. 따라서 이러한 노력을 포함하되 이에 국한되지 않고 관리자를 보호하기 위해 필요한 모든 조치를 취해야 함을 알려드립니다.

* IP 허용 목록에 추가
* [이중 인증](https://devdocs.magento.com/guides/v2.4/security/two-factor-authentication.html)
* vpn 사용
* 대신 고유한 위치 사용 `/admin`
* 올바른 암호 위생

#### 추가적인 보안 개선 사항

이 릴리스에 대한 보안 개선 사항은 기존 REST 끝점의 기본 동작이 변경되는 이 변경 사항을 포함하여 최신 보안 모범 사례에 대한 준수를 개선합니다.

**REST 및 GraphQL API를 통해 전송되는 결제 정보에 대한 기본 요금 제한**. 가맹점은 이제 REST 및 GraphQL을 사용하여 전송되는 결제 정보에 대한 요금 제한을 구성할 수 있습니다. 이렇게 추가된 보호 계층은 카드 공격의 예방을 지원하고 한 번에 여러 신용카드 번호를 테스트하는 카드 공격의 양을 잠재적으로 줄입니다. 다음을 참조하십시오 [속도 제한](https://developer.adobe.com/commerce/webapi/get-started/rate-limiting/). <!-- AC-1271 -->

의 기본 동작 [`isEmailAvailable`](https://developer.adobe.com/commerce/webapi/graphql/schema/customer/queries/is-email-available/) GraphQL 쿼리 및 ([`V1/customers/isEmailAvailable`](https://adobe-commerce.redoc.ly/2.4.6-admin/tag/customersisEmailAvailable/#operation/PostV1CustomersIsEmailAvailable)) REST 끝점이 변경되었습니다. 기본적으로 이제 API는 항상 를 반환합니다 `true`. 판매자는 원래의 비헤이비어 즉, 반환을 활성화할 수 있습니다 `true` 데이터베이스에 이메일이 없는 경우 `false` 존재하는 경우. <!-- AC-6695 -->

### 플랫폼 개선 사항

이 릴리스에 대한 플랫폼 업그레이드는 최신 보안 모범 사례를 통해 규정 준수를 개선합니다. Magento Open Source 2.4.7-Beta1은 다음을 도입합니다.

* 작성기 2.5.x. Composer 2.2.x와의 호환성이 유지됩니다.  <!-- AC-8129 -->

* **Vannish cache 7.3 지원**. 이 릴리스는 최신 버전의 Varnish Cache 7.3과 호환됩니다. 6.0.x 및 7.2.x 버전과의 호환성은 유지되지만, Magento Open Source 2.4.7-beta1을 바니시 캐시 버전 7.3 또는 버전 6.0 LTS와만 사용하는 것이 좋습니다.

* **RabbitMQ 3.11 지원**. 이 릴리스는 RabbitMQ 3.11의 최신 버전과 호환됩니다. RabbitMQ 3.9와의 호환성은 2023년 8월까지 지원되지만, RabbitMQ 3.11에서만 Magento Open Source 2.4.7-beta1을 사용하는 것이 좋습니다.

### GraphQL

이번 릴리스에서는 다음을 포함한 고객 EAV 속성에 대한 GraphQL 캐싱 기능 및 GraphQL 스키마 지원을 강화했습니다.

* **사용자 지정 특성에 대한 향상된 지원**. GraphQL 사용자 지정 특성 지원은 API 데이터를 강화하여 향상되었으므로 이제 모든 특성 유형을 지원합니다. 이제 GraphQL EAV 특성 스키마는 관리에서 고객 특성 및 고객 주소 개체를 확장하고 GraphQL을 사용하여 검색할 수 있도록 지원합니다. 구체적인 개선 영역은 다음과 같습니다. <!-- LYNX-61 110-->

   * 고객 및 고객 주소와 같은 특정 영역에 대한 사용자 지정 특성 지원 확장/추가
   * 사용자 지정 특성에 대한 캐싱이 추가되었습니다.
   * 제품에 대한 기존 사용자 정의 속성 지원 개선

* **향상된 GraphQL 캐싱 기능으로 페이지 로드 속도 향상**. 이러한 쿼리에 캐싱 기능이 추가되어 대부분의 PWA 페이지에 대한 페이지 로드 시간이 빨라집니다.

   * [`storeConfig`](https://developer.adobe.com/commerce/webapi/graphql/schema/store/queries/store-config/) <!-- B2B-2487 2258 -->
   * [`currency`](https://developer.adobe.com/commerce/webapi/graphql/schema/store/queries/currency/) <!-- B2B-2255 -->
   * [`countries`](https://developer.adobe.com/commerce/webapi/graphql/schema/store/queries/countries/) 및 [`country`](https://developer.adobe.com/commerce/webapi/graphql/schema/store/queries/country/) <!-- B2B-2256 -->
   * [`availableStores`](https://developer.adobe.com/commerce/webapi/graphql/schema/store/queries/available-stores/) <!-- B2B-2257 -->
   * [`customAttributeMetadata`](https://developer.adobe.com/commerce/webapi/graphql-api/index.html#query-customAttributeMetadata) <!-- B2B-2259 -->

* **새 GraphQL Resolver 캐시 유형** 포함:
   * CMS 페이지. CMS 페이지 확인자의 결과를 캐시합니다.  <!-- B2B-2451 -->
   * CMS 블록. 다음에서 반환된 결과를 캐시합니다. `Magento\CmsGraphQl\Model\Resolver\Blocks::resolve`. <!-- B2B-2659 -->

* 의 횟수를 줄여 GraphQL 파서의 성능이 향상되었습니다. `parse` 메서드는 요청에 따라 호출됩니다. 이제 한 번 호출됩니다. 이전에는 파서가 적어도 세 번 호출되었습니다. <!-- B2B-2606 -->

### Magento Open Source 확장 메타패키지

이 릴리스에서는 Magento Open Source 확장 메타패키지 v1.0.0-beta1을 도입했습니다. 이 메타패키지는 [결제 서비스](https://experienceleague.adobe.com/docs/commerce-merchant-services/payment-services/guide-overview.html) 이 코어 릴리스를 통한 확장. 이 메타패키지에 포함된 이 확장의 버전은 `composer update` 가 실행되어 최신 코어 릴리스로 업그레이드할 때 확장 업그레이드 프로세스가 간소화됩니다. 이 확장은 독립적인 릴리스 일정을 유지합니다.

이 확장 메타페이지의 향후 버전에는 추가 확장이 포함됩니다.

## 해결된 문제

Magento Open Source 2.4.7-베타1 코어 코드에서 수백 개의 문제를 해결했습니다. 이 릴리스에 포함된 해결된 문제의 하위 집합은 아래에 설명되어 있습니다.

### 설치, 업그레이드, 배포

<!-- ACP2E-1505 -->

* 다음 `bin/magento setup:upgrade` 다음 유형의 MySQL 메모리 제한 관련 오류로 인해 더 이상 명령이 실패하지 않습니다. `PHP Fatal error:  Allowed memory size of 4294967296 bytes exhausted (tried to allocate 20480 bytes)`. 다중 선택 속성 마이그레이션이 다음 기간 동안 메모리를 덜 사용하도록 최적화되었습니다. `setup:upgrade`.

<!-- ACP2E-1751 -->

* 이제 데이터베이스 백업 생성이 관리자 및 명령줄 모두에서 예상대로 작동합니다. 이전에는 Magento Open Source에서 다음 오류가 발생했습니다. `Syntax error or access violation: 1064 You have an error in your SQL syntax; check the manual that corresponds to your MariaDB server version for the right syntax to use near 'delimiter' at line 1, query was: delimiter ;;`.

<!-- ACP2E-1604 1605-->

* 실행 중 `setup:config:set` 을(를) 지정하지 않고 `--lock-db-prefix` 매개 변수는에서 현재 값을 더 이상 지우지 않습니다. `env.php` 파일.

<!-- ACP2E-1498 -->

* 게스트 사용자가 다른 고객 그룹과 관련된 캐시된 콘텐츠에 액세스할 수 없도록 니스 구성이 업데이트되었습니다.

<!-- https://github.com/magento/magento2/pull/33092 -->

* `bin/magento setup:upgrade` 이제 테이블 및 연결된 을 모두 설치하는 새 모듈을 설치할 때 성공적으로 완료됨 `mview` 인덱서입니다.

<!-- ENGCOM-9243 -->

* `bin/magento setup:upgrade` 이제 메시지 대기열 주제에 주제 이름이 포함되지 않은 경우 보다 자세한 오류 메시지가 표시됩니다. [GitHub-34246](https://github.com/magento/magento2/issues/34246)

<!-- https://github.com/magento/magento2/pull/36239 -->

* `bin/magento setup:upgrade` 병합된 XML 파일이 유효하지 않은 경우 보다 자세한 오류 메시지가 표시됩니다. 이제 오류 메시지에 파일 이름이 포함됩니다.

### 번들 제품

<!-- ACP2E-1478 -->

* 이제 번들 제품을 제품 옵션에서 SKU를 복제하지 않고 성공적으로 가져왔습니다. 이전에는 중복 항목이 있는 번들 제품을 가져올 때 Magento Open Source이 제품 옵션에서 여러 개의 중복 SKU를 만들었습니다.

<!-- ACP2E-1388 -->

* 이제 번들 제품 가격이 번들 내 제품 중 하나가 품절되었을 때 제대로 계산됩니다. 기존에는 번들에 포함된 상품이 품절됐다면 가격 산정에서 뺐다.

<!-- ACP2E-1503 -->

* 이제 필요한 하위 제품을 마지막으로 구매하면 번들 제품이 품절로 표시됩니다. 기존에는 단순 상품이 품절됐을 때 묶음 상품을 매장 앞에 재고로 표시했다.

<!-- ACP2E-1443 -->

* 다음을 사용하여 비필수 옵션이 있는 번들 제품을 장바구니에 추가할 때 성능 문제 발생 `addBundleProductsToCart` 돌연변이가 해결되었습니다.

<!-- ACP2E-1264 -->

* 다음 `categoryList` 이제 쿼리가 다음의 경우에 예상대로 모든 번들 옵션을 반환합니다. **[!UICONTROL Show out-of-stock products]** 구성 설정을 사용할 수 있습니다. 이전에는 품절 옵션이 쿼리 응답에 포함되지 않았습니다.

<!-- ACP2E-1345 -->

* 번들 제품에서 만들어짐 `POST V1/products` 카탈로그 가격 범위가 다음으로 설정된 경우 성공 `website`. 이전에는 무결성 제약 조건 위반이 발생했습니다. [GitHub-35595](https://github.com/magento/magento2/issues/35595)

### 캐시

<!-- ACP2E-1524 -->

* 이제 하위 제품 중 하나가 관리자에서 저장되면 상위 제품에 대해 예상대로 페이지 캐시가 지워집니다. 인덱서 상태(예약됨 또는 실시간)에 관계없이 다른 제품 유형과 유사한 구성 가능한 변형에 대한 캐시 무효화에 도달하기 위해 플러그인 검사가 도입되었습니다. 이전에는 관리자에서 구성 가능한 변형이 업데이트된 후 상점 첫 화면에서 가격이 제대로 업데이트되지 않았습니다.

<!-- ACP2E-1454 -->

* Adobe Commerce 기본 제공 UPS 통합을 사용하기 위해 새 액세스 키를 생성하려는 판매자는 현재 새 키를 생성할 수 없습니다.
