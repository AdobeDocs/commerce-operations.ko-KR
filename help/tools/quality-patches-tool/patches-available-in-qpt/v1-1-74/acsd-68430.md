# ACSD-68430: 레코드에 속성 구성에서 삭제된 여러 속성 옵션이 포함된 경우 고객 또는 고객 주소를 저장하지 못했습니다

ACSD-68430 패치는 레코드에 속성 구성에서 삭제된 여러 속성 옵션이 포함된 경우 고객 또는 고객 주소를 저장하지 못하는 문제를 수정합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.74가 설치되어 있을 때 사용할 수 있습니다. 패치 ID는 ACSD-68430입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.7-p6

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.6 - 2.4.7-p8

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ko). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

속성 구성에서 삭제된 여러 속성 옵션이 포함된 고객 또는 고객 주소를 저장할 때 작업이 실패합니다.

<u>재현 단계</u>:

1. 관리자로 이동하여 옵션 A, B, C, D를 사용하여 다중 선택 사용자 지정 고객 속성(예: &quot;test&quot;)을 만듭니다.
1. 관리에서 고객을 생성하고 이 속성에 대한 값을 저장합니다(예: A, B, C 선택).
1. 속성 구성으로 돌아가서 고객에게 저장된 옵션 중 하나를 삭제합니다(예: 삭제 옵션 C).
1. 관리에서 동일한 고객의 레코드를 편집해 보세요. 성별 또는 접두사 등의 필드를 변경하고 **[!UICONTROL Save]**&#x200B;을(를) 클릭합니다.

<u>예상 결과</u>:

고객 레코드 변경 사항은 오류 없이 저장됩니다.

<u>실제 결과</u>:

고객 레코드에 변경 사항을 저장할 때 오류가 발생했습니다. *특성 [attribute_name]에 ID가 [deleted_option_id]*&#x200B;인 옵션이 없습니다.
