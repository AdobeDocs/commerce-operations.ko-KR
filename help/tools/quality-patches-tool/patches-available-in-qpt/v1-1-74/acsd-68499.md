# ACSD-68499: GraphQL updateCartItems 돌연변이는 사용 가능한 재고 이상의 수량에 대해 *성공*&#x200B;을 반환합니다.

ACSD-68499 패치는 사용 가능한 재고를 초과하는 수량을 업데이트할 때 GraphQL `updateCartItems` 돌연변이가 잘못된 *성공* 응답을 반환하여 수량 및 합계가 부풀려지는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.74가 설치되어 있을 때 사용할 수 있습니다. 패치 ID는 ACSD-68499입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p2

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ko). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

Adobe Commerce의 GraphQL `updateCartItems` 돌연변이는 사용 가능한 재고 이상의 수량을 업데이트할 때 *성공* 응답을 잘못 반환하여 장바구니 합계 및 수량이 부풀려졌습니다.

<u>재현 단계</u>:

1. 빈 장바구니를 만듭니다.
1. 재고가 제한된 제품(예: 10개 사용 가능)을 장바구니에 추가합니다.
1. 장바구니를 쿼리하고 `cart_item_id`을(를) 확인합니다.
1. 다음 돌연변이를 사용하여 장바구니 품목 수량을 100(가용 재고 10개를 초과)으로 갱신합니다.

   ```
   mutation ($cart: String!, $cartItemId: Int!, $quantity: Float!) {
     updateCartItems(
       input: {
         cart_id: $cart
         cart_items: [{ cart_item_id: $cartItemId, quantity: $quantity }]
       }
     ) {
       cart {
         items {
           product {
             name
           }
           quantity
         }
         prices {
           grand_total {
             value
             currency
           }
         }
       }
     }
   }
   ```

1. 장바구니를 다시 질의하여 실제 수량을 확인합니다.

<u>예상 결과</u>:

재고 유효성 검사로 인해 부적합한 수량 업데이트가 방지됩니다.

<u>실제 결과</u>:

`updateCartItems` 돌연변이는 100의 수량으로 *성공* 응답을 반환합니다.
