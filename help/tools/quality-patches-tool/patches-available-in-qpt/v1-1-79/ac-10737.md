# AC-10737: `bin/magento setup:db:status` 명령이 JSON 데이터 형식을 인식하지 않습니다.

AC-10737 패치는 `bin/magento setup:db:status` 명령이 JSON 데이터 형식을 인식하지 못하는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79가 설치된 경우에 사용할 수 있습니다. 패치 ID는 AC-10737입니다. 이 문제는 Adobe Commerce 2.4.7-p5에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.6-p15

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.4 - 2.4.7-p4

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ko). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

`bin/magento setup:db:status` 명령이 JSON 열 유형을 인식하지 못하며 선언적 스키마가 최신 상태가 아니라고 잘못 보고합니다.

<u>재현 단계</u>:

1. `db_schema.xml`에 `JSON` 열이 있는 새 테이블을 정의하는 사용자 지정 모듈을 만드십시오. 예:

   ```
   <table name="example_table" engine="innodb" resource="default">
       <column xsi:type="int" name="entity_id" identity="true" nullable="false" unsigned="true"/>
       <column xsi:type="json" name="content" nullable="false"/>
       <constraint referenceId="PRIMARY" xsi:type="primary">
           <column name="feed_missing_id"/>
       </constraint>
   </table>
   ```

1. 모듈에 대한 허용 목록을 생성합니다.

   ```
   bin/magento setup:db-declaration:generate-whitelist --module-name Sample_Module
   ```

1. 데이터베이스 스키마를 업그레이드합니다.

   ```
   bin/magento setup:upgrade
   ```

1. 선언적 스키마 상태를 확인합니다.

   ```
   bin/magento setup:db:status
   ```

1. 이 명령은 선언적 스키마를 최신 상태가 아닌 것으로 보고합니다.



<u>예상 결과</u>:

1. `setup:db:status` 명령은 선언적 스키마가 최신임을 보고합니다.

<u>실제 결과</u>:

1. 이 명령은 선언적 스키마가 최신 상태가 아님을 보고하고 다음 메시지를 표시합니다.

```
Declarative Schema is not up to date
Run 'setup:upgrade' to update your DB schema and data.
```
