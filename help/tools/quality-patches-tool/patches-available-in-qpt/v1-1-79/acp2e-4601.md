# ACP2E-4601: 결제 권한 부여 트랜잭션 자체 참조 `parent_id`을(를) 다시 저장하고 무한 루프/OOM을 트리거합니다.

ACP2E-4601 패치는 `parent_txn_id` 없이 권한 부여 트랜잭션을 다시 저장할 때 결제 트랜잭션 처리가 비효율적으로 작동하는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79가 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4601입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p3

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ko). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

`parent_txn_id` 없이 만들어진 인증 거래를 다시 저장할 때 결제 거래 처리가 비효율적으로 동작합니다.

<u>재현 단계</u>:

1. 온라인 결제 방법을 설정합니다.
1. `parent_txn_id` 없이 권한 부여 결제 거래를 만드는 주문을 합니다.
1. 인증 트랜잭션을 다시 저장합니다.

<u>예상 결과</u>:

결제 거래 처리가 완료되었습니다.

<u>실제 결과</u>:

인증 트랜잭션 재저장에 실패했습니다.

* Adobe Commerce 또는 Magento Open Source 온-프레미스: [!DNL Quality Patches Tool] 가이드의 [[!DNL Quality Patches Tool] > 사용량](/help/tools/quality-patches-tool/usage.md)
