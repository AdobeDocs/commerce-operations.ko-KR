# ACP2E-4531: 하위 페이지 URL 키가 변경될 때 CMS 계층에서 계층 URL을 업데이트하지 않습니다

ACP2E-4531 패치는 CMS 페이지의 URL 키를 변경해도 페이지의 계층 URL이 업데이트되지 않는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79가 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4531입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p3

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

CMS 페이지의 URL 키를 변경해도 페이지의 계층적 URL은 업데이트되지 않습니다.

<u>재현 단계</u>:

1. CMS 페이지를 만듭니다(예: *도움말*).
1. 두 번째 CMS 페이지(예: *my-account*)를 만들고 *help*&#x200B;을(를) 계층 구조에서 상위로 설정합니다.
1. Storefront에서 두 페이지를 모두 로드하면 두 페이지가 올바르게 확인됩니다.
1. 내 계정 페이지의 URL 키(*검색 엔진 최적화* 아래)를 *my-account-2*(으)로 변경하십시오.
1. Storefront에서 `/my-account-2` 및 `/help/my-account-2`을(를) 로드합니다.

<u>예상 결과</u>:

CMS 페이지 URL 키가 변경되면 계층 URL이 그에 따라 업데이트됩니다. `/help/my-account-2`이(가) 페이지로 확인되고 `/my-account`이(가) 더 이상 확인되지 않습니다(이러한 URL이 존재하지 않음).

<u>실제 결과</u>:

`/my-account-2`이(가) `/my-account`(으)로 리디렉션하고 `/help/my-account-2`이(가) 404 오류를 반환합니다. 하위 페이지 URL 키가 변경되면 계층 URL이 업데이트되지 않습니다.

개별 패치를 적용하려면 배포 방법에 따라 다음 링크를 사용합니다.

* Adobe Commerce 또는 Magento Open Source 온-프레미스: [!DNL Quality Patches Tool] 가이드의 [[!DNL Quality Patches Tool] > 사용량](/help/tools/quality-patches-tool/usage.md)
