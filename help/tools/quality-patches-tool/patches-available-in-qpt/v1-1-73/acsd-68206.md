# ACSD-68206: [!DNL GraphQL] 서버가 속도 제한과 PHP Redis 확장으로 실패합니다.

ACSD-68206 패치는 속도 제한 기능 및 PHP Redis 확장 기능을 사용할 때 GraphQL Application Server에서 오류가 발생하여 속도 제한 기능이 중단되는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACSD-68206입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p2

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ko). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

이 문제는 GraphQL 애플리케이션 서버가 PHP Redis 확장이 설치된 상태에서 속도 제한 기능을 사용하여 속도 제한 기능이 손상될 때 발생합니다.

<u>재현 단계</u>:

1. PHP Redis 확장을 설치합니다.
1. 애플리케이션에 대한 Redis 구성.
1. 요금 제한기를 설정하여 매 분마다 고객당 한 개의 주문을 허용합니다.
1. [!DNL GraphQL] 응용 프로그램 서버를 시작합니다.
1. 고객은 `PlaceOrder` [!DNL GraphQL] 돌연변이를 사용하여 주문을 제출합니다.
1. 1분 내에 동일한 고객에 대해 `PlaceOrder` [!DNL GraphQL] 돌연변이를 사용하는 두 번째 주문을 제출합니다.

<u>예상 결과</u>:

`PlaceOrder` [!DNL GraphQL] 요청이 순서 시도를 제한하는 메시지를 반환합니다.

<u>실제 결과</u>:

서버가 두 번째 PlaceOrder GraphQL 요청에 대해 500 오류를 반환합니다. 예외 로그 레코드:

```
report.CRITICAL: Error: Call to a member function exec() on null in vendor/colinmollenhour/credis/Client.php:1336
```
