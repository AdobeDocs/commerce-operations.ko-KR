# ACSD-67908: JavaScript 파일이 다중 스토어 설정에서 올바르게 병합되지 않음

ACSD-67908 패치는 JavaScript 파일이 다중 스토어 설정에서 올바르게 병합되지 않는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACSD-67908입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.7-p5

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

병합된 JavaScript 파일이 다중 스토어 설정에서 올바르게 생성되지 않아 파일이 손상되고 상점 오류가 발생합니다.

<u>재현 단계</u>:

1. 50개 이상의 웹 사이트, 스토어 및 스토어 조회수를 설정합니다.
1. 프로덕션 모드를 설정하고 컴파일을 건너뜁니다.

   ```
   bin/magento deploy:mode:set production --skip-compilation
   ```

1. 생성된 코드 배포:

   ```
   bin/magento setup:di:compile
   ```

1. 컴팩트 전략을 사용하여 정적 콘텐츠 배포:

   ```
   bin/magento setup:static-content:deploy --strategy compact --jobs 10
   ```

1. 병합된 캐시 폴더의 내용을 삭제합니다.

   ```
   rm -rf pub/static/_cache/merged/*
   ```

1. 캐시 지우기:

   ```
   bin/magento cache:clean
   ```

1. 별도의 브라우저 탭 또는 창에서 모든 스토어 보기를 동시에 엽니다.


<u>예상 결과</u>:

모든 JavaScript 파일이 정상적으로 병합되고 상점 첫 화면의 오류 없이 작동합니다.

<u>실제 결과</u>:

병합된 일부 JavaScript 파일이 손상되어 올바른 RequireJS 모듈 선언으로 시작하지 않습니다.
