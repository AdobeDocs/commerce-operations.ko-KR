# ACSD-67171: 체크아웃 중에 만료된 [!DNL B2B] 세션이 로그인 대신 액세스 거부 페이지로 리디렉션됩니다.

ACSD-67171 패치는 세션이 만료되거나 체크 아웃 중에 제거될 때 [!DNL B2B] 고객이 *액세스 거부됨* 페이지로 리디렉션되는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACSD-67171입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p1

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

[!DNL B2B] 사용자의 세션이 만료되거나 제거되면 체크 아웃 중에 고객 로그인 페이지로 리디렉션되지 않고 액세스 거부 페이지가 표시됩니다.

<u>재현 단계</u>:

1. 관리 패널에서 [!UICONTROL Stores] > [!UICONTROL Configuration] > [!UICONTROL General] > [!UICONTROL [!DNL B2B] Features]&#x200B;(으)로 이동하고 [!UICONTROL [!DNL B2B] Features]을(를) *예*(으)로 설정한 다음 [!UICONTROL Customers] > [!UICONTROL Persistent Shopping Cart]&#x200B;(으)로 이동하고 [!UICONTROL Enable Persistence]을(를) *예*(으)로 설정합니다.
1. 새 [!UICONTROL Customers] 회사를 만들려면 [!UICONTROL Companies] > [!UICONTROL Add New Company]&#x200B;(으)로 이동하고 [!DNL B2B]을(를) 선택하십시오.
1. [!DNL B2B] 회사 사용자로 상점 앞에 로그인합니다.
1. 장바구니에 제품을 추가합니다.
1. 체크아웃 페이지로 이동합니다.
1. 세션이 만료되도록 허용합니다.
1. 체크아웃 페이지를 다시 로드합니다.

<u>예상 결과</u>:

고객 로그인 페이지로 리디렉션됩니다.

<u>실제 결과</u>:

`/company/accessdenied`(으)로 리디렉션되고 *이 페이지를 볼 수 있는 권한이 없습니다. 이 오류가 있다고 생각되면 회사 관리자에게 문의하십시오.*
