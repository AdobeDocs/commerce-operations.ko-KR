# ACP2E-4653: [!DNL REST] API에 장바구니 가격 규칙 범주 범위 조건이 없습니다.

ACP2E-4653 패치는 [!DNL REST] API를 통해 규칙을 검색하거나 업데이트할 때 **[!UICONTROL Category (Parent Only)]** 및 **[!UICONTROL Category (Children Only)]**&#x200B;에 대한 장바구니 가격 규칙 조건 특성 범위가 노출되지 않는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4653입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p2

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ko). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

장바구니 가격 규칙이 [!DNL REST] API를 통해 검색되거나 업데이트되면 **[!UICONTROL Category (Parent Only)]** 및 **[!UICONTROL Category (Children Only)]**&#x200B;에 대한 특성 범위가 노출되지 않습니다.

<u>재현 단계</u>:

1. 관리자 자격 증명으로 `rest/V1/integration/admin/token`에 `POST` 요청을 보내 관리자 토큰을 생성합니다.
1. `category_ids` 특성을 사용하는 작업 조건으로 [!DNL REST] 끝점 `rest/V1/salesRules`을(를) 사용하여 장바구니 가격 규칙을 만듭니다.
1. 관리자 패널에서 장바구니 가격 규칙을 엽니다. **[!UICONTROL Actions]** 섹션의 **[!UICONTROL Apply the rule only to cart items matching the following conditions (leave blank for all items]**&#x200B;에서 **[!UICONTROL Category (Children Only)]** 조건을 수동으로 추가합니다.
1. **[!UICONTROL Category (Children Only)]**&#x200B;이(가) `<category_id>`이(가) 되지 않도록 조건을 설정하십시오.
1. `GET` 요청을 `rest/V1/salesRules/<rule_id>`(으)로 보내 API를 통해 규칙을 검색합니다.
1. 응답에서 반환된 `action_condition` 값을 확인합니다.

<u>예상 결과</u>:

1. [!DNL REST] API를 통해 **[!UICONTROL Category (Children Only)]** 조건으로 장바구니 가격 규칙을 만들고, 검색하고, 업데이트할 수 있어야 합니다.
1. 범주, **[!UICONTROL Category (Parent Only)]** 및 **[!UICONTROL Category (Children Only)]**&#x200B;을(를) 올바르게 구분할 수 있도록 특성 범위를 명시적으로 노출해야 합니다. 예를 들어 `action_condition`은(는) 다음과 같이 반환되어야 합니다.

   ```
           "action_condition": {
               "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product\\Combine",
               "conditions": [
                   {
                       "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product",
                       "operator": "!=",
                       "attribute_name": "category_ids",
                       "value": "4",
                       "extension_attributes": {
                           "attribute_scope": ""
                       }
                   },
                   {
                       "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product",
                       "operator": "==",
                       "attribute_name": "category_ids",
                       "value": "5",
                       "extension_attributes": {
                           "attribute_scope": "children"
                       }
                   }
               ],
               "aggregator_type": "all",
               "operator": null,
               "value": "1"
           },
   ```

<u>실제 결과</u>:

1. 두 조건 특성 이름이 모두 `category_ids`(으)로 반환됩니다.
1. [!DNL REST] API를 통해 **[!UICONTROL Category (Children Only)]** 조건으로 규칙을 만들거나 업데이트하는 방법을 알 수 없습니다.
