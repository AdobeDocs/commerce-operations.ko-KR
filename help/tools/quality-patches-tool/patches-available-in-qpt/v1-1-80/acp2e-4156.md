# ACP2E-4156: [!DNL REST] API 배송 주소 유효성 검사가 관리자 특성 구성을 무시합니다.

ACP2E-4156 패치는 [!DNL REST] API의 배송 주소 유효성 검사가 Admin에 정의된 특성 구성을 준수하지 않는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4156입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p4

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ko). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

[!DNL REST] API에서 배송 주소 유효성 검사가 Admin에 구성된 고객 주소 특성 설정을 준수하지 않습니다.

<u>재현 단계</u>:

1. 관리 패널에서 **[!UICONTROL Stores]** > **[!UICONTROL Attributes]** > **[!UICONTROL Customer Address]**(으)로 이동합니다.
1. 특성 코드 *firstname*(으)로 특성을 엽니다.
1. **[!UICONTROL Attribute Properties]**&#x200B;에서 다음을 설정합니다.
   1. **[!UICONTROL Input Validation]**: 길이만.
   1. **[!UICONTROL Minimum Text Length]**: *1*.
   1. **[!UICONTROL Maximum Text Length]**: *20*.
1. 상점으로 이동하여 테스트 제품을 장바구니에 추가합니다.
1. 20자를 초과하는 *firstname* 값으로 배송 정보를 추가하거나 업데이트하려면 [!DNL REST] API 요청을 `/rest/default/V1/guest-carts/cartId/shipping-information`에 보내십시오.

<u>예상 결과</u>:

*firstname* 값이 구성된 길이 제한을 초과함을 나타내는 오류가 반환됩니다.

<u>실제 결과</u>:

장바구니 배송 정보가 [!DNL REST] API 요청에서 보낸 *firstname* 값으로 업데이트되었습니다.
