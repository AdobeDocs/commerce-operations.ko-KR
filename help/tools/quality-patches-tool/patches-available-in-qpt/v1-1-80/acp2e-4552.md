# ACP2E-4552: GraphQL 응답이 회사 상태를 반환하지 않음

ACP2E-4552 패치는 GraphQL 응답에서 회사 상태가 반환되지 않는 문제를 수정합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4552입니다. 이 문제는 Adobe Commerce B2B 1.5.2-p3에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) B2B 1.5.2-p3

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) B2B >=1.5.0 &lt;1.5.3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

GraphQL 응답은 회사 상태를 반환하지 않습니다.

<u>필수 구성 요소</u>:

[!DNL B2B]이(가) 설치되었습니다.

<u>재현 단계</u>:

1. 관리자에서 **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]**(으)로 이동한 다음 **[!UICONTROL Enable Company]**&#x200B;을(를) `Yes`(으)로 설정합니다.
1. 관리자로 Commerce 관리자에 로그인합니다.
1. 활성, 거부됨 및 승인 보류 중 상태의 세 회사를 만듭니다.
1. 관리자에서 **[!UICONTROL Customers]** > **[!UICONTROL All Customers]**(으)로 이동합니다. 고객 계정을 열고 **[!UICONTROL Associate to Company]** 목록에서 고객을 만든 각 회사에 할당합니다.
1. 다음 GraphQL 요청 보내기:

   ```
   {
     customer {
       firstname
       lastname
       email
       status
       companies {
        items {
           id
           legal_name
           name
           status
        }
       }
     }
   }
   ```

<u>예상 결과</u>:

GraphQL 응답은 고객과 연결된 모든 회사에 대해 올바른 상태 값(예: 활성, 승인 보류, 거부됨 및 차단됨)을 반환합니다. 회사 상태별로 응답을 필터링하거나 정렬할 수도 있습니다.

<u>실제 결과</u>:

GraphQL 응답에는 고객과 연관된 회사의 회사 상태 값이 포함되지 않습니다.

## 패치 적용

개별 패치를 적용하려면 배포 방법에 따라 다음 링크를 사용합니다.

* Adobe Commerce 또는 Magento Open Source 온-프레미스: [!DNL Quality Patches Tool] 가이드의 [[!DNL Quality Patches Tool] > 사용량](/help/tools/quality-patches-tool/usage.md)
