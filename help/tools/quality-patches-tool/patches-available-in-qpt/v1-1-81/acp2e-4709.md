# ACP2E-4709: 페이지 계층에 큰따옴표가 포함된 경우 CMS 페이지가 페이지 빌더에 저장되지 않습니다

ACP2E-4709 패치는 페이지 계층에 큰따옴표가 포함된 경우 CMS 페이지가 페이지 빌더에 저장되지 않는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.81이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4709입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.7-p9

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.7 - 2.4.8-p5

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ko). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

페이지 계층에 큰따옴표가 포함된 경우 CMS 페이지가 Page Builder에 저장되지 않습니다.

<u>필수 구성 요소</u>:

페이지 빌더가 활성화되어 있습니다.

<u>재현 단계</u>:

1. 관리 사이드바에서 **[!UICONTROL Stores]** > **설정** > **구성**(으)로 이동합니다. **[!UICONTROL General]** 아래의 왼쪽 패널에서 **콘텐츠 관리**&#x200B;를 선택합니다.
1. **[!UICONTROL CMS Page Hierarchy]**&#x200B;을(를) 확장하고 **[!UICONTROL Enable Hierarchy Functionality]**&#x200B;을(를) **[!UICONTROL Yes]**(으)로 설정한 다음 **[!UICONTROL Save Config]**&#x200B;을(를) 클릭합니다.
1. 관리 사이드바에서 **컨텐츠** > **[!UICONTROL Elements]** > **[!UICONTROL Pages]**(으)로 이동한 다음 **[!UICONTROL Add New Page]**&#x200B;을(를) 클릭합니다.
1. 큰따옴표가 포함된 페이지 제목을 입력하고 **계층** 탭의 웹 사이트 루트 아래에 페이지를 놓은 다음 **[!UICONTROL Save]**&#x200B;을(를) 클릭합니다.
1. Page Builder를 사용하여 페이지에 콘텐츠를 추가하고 **[!UICONTROL Save]**&#x200B;을(를) 클릭합니다.

<u>예상 결과</u>:

새 CMS 페이지가 오류 없이 성공적으로 저장됩니다.

<u>실제 결과</u>:

&#x200B;- 새 CMS 페이지가 저장되지 않습니다.
&#x200B;- 브라우저 콘솔에 `Uncaught SyntaxError: JSON.parse: expected ',' or '}' after property value in object at line 1 column 79 of the JSON data` 오류가 표시됩니다.
