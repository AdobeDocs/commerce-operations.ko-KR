# ACSD-69129: 기본 기본 웹 사이트를 삭제하고 보조 웹 사이트를 기본값으로 사용한 후 [!DNL REST] API 계층 가격 업데이트에 실패했습니다.

ACSD-69129 패치는 [!DNL REST] API를 통해 보조 웹 사이트의 계층 가격을 업데이트하려고 할 때 기본 기본 웹 사이트를 삭제하고 보조 웹 사이트를 기본값으로 사용하는 문제가 오류가 발생하는 문제를 해결했습니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACSD-69129입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.6-p13, 2.4.8-p3

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.5 - 2.4.6-p13, 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ko). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

기본 기본 웹 사이트를 삭제하고 보조 웹 사이트를 기본값으로 사용하면 [!DNL REST] API를 통해 보조 웹 사이트의 계층 가격을 업데이트하려고 할 때 오류가 발생합니다.

<u>재현 단계</u>:

1. [!UICONTROL Admin] 패널에 로그인합니다.
1. **[!UICONTROL Catalog Price Scope]** = *웹 사이트*&#x200B;를 설정합니다.
1. 새 웹 사이트/스토어/스토어를 만듭니다.
1. 새로 만든 웹 사이트에 대해 *기본값으로 설정*&#x200B;을 설정합니다.
1. 캐시를 지웁니다.
1. 새 제품을 만듭니다. 두 번째 웹 사이트에만 할당합니다.
1. 두 번째 웹 사이트의 계층 가격을 설정하려면 [!DNL REST] API 요청을 전송하십시오.
1. 결과는 *200* 응답이 있는 빈 배열이어야 합니다.
1. [!UICONTROL Admin]에서 *기본 웹 사이트*&#x200B;를 삭제합니다.
1. 두 번째 웹 사이트의 계층 가격을 설정하려면 [!DNL REST] API 요청을 전송하십시오.

<u>예상 결과</u>:

* *200* 응답을 받았습니다.
* 해당 제품의 계층 가격이 업데이트되었습니다.

<u>실제 결과</u>:

* *잘못된 특성 웹 사이트 ID = %websiteId. 행 ID: SKU = %SKU, 웹 사이트 ID: %websiteId, 고객 그룹: %customerGroup, 수량: %qty.응답에* 오류가 반환되었습니다.
* 해당 제품의 계층 가격은 업데이트되지 않습니다.
