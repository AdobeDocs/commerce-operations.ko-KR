# ACSD-69308: `special_price`이(가) 웹 사이트 수준에서 설정된 경우 카탈로그 가격 규칙이 적용되지 않습니다.

ACSD-69308 패치는 `special_price`이(가) 웹 사이트 수준(&quot;모든 스토어 보기&quot;가 아님)에서만 설정되었을 때 카탈로그 가격 규칙이 적용되지 않던 문제를 해결합니다. 수정 후에는 먼저 웹 사이트의 기본 스토어를 확인하여 카탈로그 가격 규칙이 올바르게 적용됩니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACSD-69308입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

`special_price`이(가) 웹 사이트 수준에서만 구성된 경우 카탈로그 가격 규칙이 적용되지 않았습니다. 시스템이 특정 매장 보기에 지정된 제품에 대한 가격을 올바르게 인식하지 못하여 가격이 일관되지 않았습니다.

<u>재현 단계</u>:

1. 관리자 패널에 로그인합니다.
1. 사용자 지정 웹 사이트, 스토어 및 스토어 보기(web2)를 만듭니다.
1. **[!UICONTROL Catalog Price Scope]**&#x200B;을(를) *웹 사이트*(으)로 설정합니다.
1. **[!UICONTROL Use for Promo Rule Conditions]** 특성에 대해 *을(를)*&#x200B;예`special_price`(으)로 설정합니다.
1. 다음 카탈로그 가격 규칙을 만듭니다.

```
Website: web2
Conditions:
Special Price  greater than  15
Special Price  less than  50
Actions:
Apply: Apply as percentage of original
Discount Amount: 100
```

1. **web2** 스토어 보기에만 할당된 두 제품을 만듭니다.

제품 1:

```
Set Advanced Pricing:
All Store View Scope: 30.00
web2 scope: 40.00
```

제품 2:

```
Set Advanced Pricing:
All Store View Scope: keep empty
web2 scope: 40.00
```

1. **web2** 스토어 보기 아래의 상점에서 각 제품을 다시 인덱싱하고 로드합니다.

<u>예상 결과</u>:

웹2 스토어 보기에 카탈로그 가격 규칙을 적용해야 합니다. 제품 02의 특별 가격이 카탈로그 규칙 조건과 일치하므로 *0.00* 가격을 표시해야 합니다.

<u>실제 결과</u>:

* 제품 01: 가격이 카탈로그 규칙 조건과 일치하는 *0.00*(으)로 표시됩니다.
* 제품 02: 카탈로그 규칙이 웹 사이트 수준에서 설정된 특별 가격에 따라 적용되어야 하므로 가격이 *40.00*(으)로 표시되는데, **예상하지 않음**.
