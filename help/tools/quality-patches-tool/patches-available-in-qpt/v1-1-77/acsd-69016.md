# ACSD-69016: 시간대가 다른 웹 사이트에는 특별 가격이 적용되지 않습니다

ACSD-69016 패치는 코드 기본값이 있는 스토어가 존재하지 않는 경우 제품 업데이트 [!DNL REST] API 요청이 실패하는 문제를 해결합니다. 이로 인해 다른 시간대에 만들어진 웹 사이트에 대해 특별 가격이 적용되지 않습니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACSD-69016입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p1

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

코드 기본값이 있는 스토어가 없으면 제품 업데이트 [!DNL REST] API 요청이 실패하여 다른 시간대에 만들어진 웹 사이트에 대해 특별 가격이 적용되지 않습니다.

<u>재현 단계</u>:

1. **[!UICONTROL Admin panel]**&#x200B;에서 **[!UICONTROL Stores]** > **[!UICONTROL &#x200B; All Stores]** > **[!UICONTROL Default Store View]**(으)로 이동합니다.
1. 기본 저장소 보기의 코드 값을 변경합니다.
1. 제품을 업데이트하려면 [!DNL REST] API 요청을 보냅니다.

<u>예상 결과</u>:

요청이 성공적으로 완료되고 제품이 업데이트됩니다.

<u>실제 결과</u>:

요청이 실패하고 제품이 업데이트되지 않습니다.
