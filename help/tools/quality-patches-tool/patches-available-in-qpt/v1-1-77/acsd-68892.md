# ACSD-68892: 캐시 가능한 페이지에 대해 일관되지 않은 [!DNL Fastly] 캐싱 동작

ACSD-68892 패치는 캐시 가능한 페이지가 [!DNL Fastly] 캐시에서 제대로 저장되거나 제공되지 않아 캐싱 동작이 일관되지 않고 성능이 저하되는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACSD-68892입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p2

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

캐시 가능한 페이지가 [!DNL Fastly] 캐시에서 제대로 저장되거나 제공되어 캐싱 동작이 일관되지 않고 성능이 저하됩니다.

<u>재현 단계</u>:

1. [!DNL Fastly] 캐시를 사용하도록 설정합니다.
1. 새 웹 사이트/스토어/스토어 보기를 만듭니다.
1. `index.php`에 보조 웹 사이트 코드 추가:

```
$_SERVER['MAGE_RUN_TYPE'] = 'website';
$_SERVER['MAGE_RUN_CODE'] = 'web2';
```

1. curl을 사용하여 캐시 제어 헤더 확인:

```
curl -Is https://localhost/m247/pub/ | grep cache-control
```

<u>예상 결과</u>:

캐시 가능한 페이지가 [!DNL Fastly]에 캐시됩니다.

<u>실제 결과</u>:

캐시 가능한 페이지가 [!DNL Fastly]에서 캐시되지 않습니다. `max-age=0`이(가) 헤더에서 반환되었습니다.

```
curl -Is https://localhost/m24develop/pub/ |grep cache-control
cache-control: max-age=0, must-revalidate, no-cache, no-store
```

* Adobe Commerce 또는 Magento Open Source 온-프레미스: [[!DNL Quality Patches Tool]  가이드의 ](/help/tools/quality-patches-tool/usage.md)> 사용량[!DNL Quality Patches Tool]
