# ACSD-63687: [!DNL Redis] 캐시 정리 문제로 인해 잘못된 가격이 표시됩니다.

ACSD-63687 패치는 [!DNL Redis] 캐시 가비지 수집기가 LUA 모드 없이 실행되어 경합 상태가 발생할 때 잘못된 가격이 표시되는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACSD-63687입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.5-p9

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.5 - 2.4.6-p13

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

[!DNL Redis] 캐시 정리에 실패하여 잘못된 가격이 표시됩니다.

<u>재현 단계</u>:

1. Adobe Commerce 환경에 대해 [!DNL Redis]을(를) 구성합니다.
1. 상점 첫 화면에서 여러 제품 카테고리를 만들 수 있습니다.
1. 범주 페이지에 반복 요청을 보내는 스크립트를 만듭니다. JMeter 스크립트 또는 이와 동등한 로드 테스트 도구를 사용합니다.
1. 스크립트가 범주 페이지에 요청을 보내는 동안 `block_html` 캐시를 지우고 `backend_clean_cache` cron 작업을 여러 번 실행합니다.
1. 제공된 스크립트를 실행하여 MAGE 캐시 집합에 존재하지 않는 [!DNL Redis]개의 캐시 키 레코드를 식별하십시오. 로컬 프로젝트 구성과 일치하도록 [!DNL Redis] 포트 및 MAGE 캐시 태그 접두사를 업데이트합니다.

   ```
   #!/bin/bash
   
   REDIS_PORT=6379
   REDIS_DB=0
   SET_NAME="zc:ti:af6_MAGE"
   
   redis-cli -p $REDIS_PORT -n $REDIS_DB keys "zc:k:*" | while read KEY; do
     STRIPPED_KEY=${KEY#"zc:k:"}
     EXISTS=$(redis-cli -p $REDIS_PORT -n $REDIS_DB sismember "$SET_NAME" "$STRIPPED_KEY")
   
     if [ "$EXISTS" -eq 0 ]; then
       TIMESTAMP=$(redis-cli -p $REDIS_PORT -n $REDIS_DB hget "$KEY" "m")
   
       if [ -n "$TIMESTAMP" ]; then
         FORMATTED_TIME=$(date -r "$TIMESTAMP" +"%y-%m-%d %H:%M:%S")
         echo "$KEY - m: $FORMATTED_TIME"
       else
         echo "$KEY - m: (null)"
       fi
     fi
   done
   ```

<u>예상 결과</u>:

해당 레코드가 없는 캐시 키가 [!DNL Redis]에 만들어지지 않으며 `backend_clean_cache` Cron 작업이 캐시 항목을 올바르게 정리합니다.

<u>실제 결과</u>:

캐시 키가 해당 레코드 없이 [!DNL Redis]에 생성되므로 `backend_clean_cache` 크론으로 트리거된 캐시 정리가 작동하지 않습니다.
