# ACP2E-4456: GraphQL 돌연변이가 있는 주문을 취소하면 기프트 카드로 완전히 결제된 주문이 마감됨 상태로 이동하지 않습니다

ACP2E-4456 패치는 GraphQL 주문 취소가 기프트 카드 결제 주문을 [!UICONTROL Closed] 상태로 전환하지 않는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4456입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p3

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

GraphQL 돌연변이를 통해 주문을 취소하면 기프트 카드 결제 주문이 [!UICONTROL Closed] 상태로 전환되지 않는 문제가 발생합니다.

<u>재현 단계</u>:

1. 모든 유형의 기프트 카드를 만듭니다.
1. 간단한 제품을 만듭니다.
1. **[!UICONTROL Free Shipping]** 사용
1. **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL Sales]** > **[!UICONTROL Order cancellation]**(으)로 이동하여 GraphQL 주문 취소를 활성화하고 **[!UICONTROL Order cancellation through GraphQL]**&#x200B;을(를) *예*(으)로 설정합니다.
1. 상점 계정에 로그인합니다.
1. 기프트 카드를 구입하고 주문에 대한 송장을 생성한 다음 기프트 카드 번호를 기록합니다.
1. 간단한 제품을 장바구니에 추가합니다.
1. 체크아웃 중에 **[!UICONTROL Free Shipping]**&#x200B;을(를) 선택하십시오.
1. 기프트 카드로 결제해 주세요.
1. 주문에 대한 송장을 발행합니다.
1. 다음 GraphQL 돌연변이를 사용하여 주문을 취소합니다.

   ```
   mutation {
   	cancelOrder(input: { order_id: "Nw==", reason: "Other" }) {
   		error
   		order {
   			carrier
   			created_at
   			email
   			gift_receipt_included
   			grand_total
   			id
   			increment_id
   			number
   			order_date
   			shipping_method
   			status
   		}
   	}
   }
   ```

<u>예상 결과</u>:

주문 상태가 취소됨으로 변경됩니다.

<u>실제 결과</u>:

주문이 취소되지 않았습니다
