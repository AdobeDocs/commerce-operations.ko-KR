# ACP2E-4591: &quot;최초 구매자&quot;와 같이 주문 수에 따른 고객 세그먼트는 REST API를 통해 주문된 주문에 대해 업데이트하지 않습니다

ACP2E-4591 패치는 &quot;최초 구매자&quot;와 같이 주문 수를 기반으로 하는 고객 세그먼트가 고객 세션이 없어서 REST API를 통해 주문이 들어왔을 때 업데이트되지 않는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4591입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.7-p4

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.4 - 2.4.7-p9

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

주문 수를 기반으로 하는 고객 세그먼트(예: &quot;처음 구매자&quot;는 REST API를 통해 주문이 들어올 때 업데이트되지 않습니다.

<u>재현 단계</u>:

1. 다음 조건을 사용하여 *첫 번째 사용자*(이)라는 고객 세그먼트를 만듭니다.

   ```
   If ALL of these conditions are TRUE:
   Total Number of Orders less than 1 while ALL of these Conditions match:
   Order Status is Any
   ```

1. 고객을 만듭니다.
1. `matchCustomerSegmentProcessor` 소비자를 실행하고 새 고객이 **[!UICONTROL Matched Registered Customers]** 표에 표시되는지 확인하십시오.
1. 고객으로 로그인하고 상점 첫 화면에서 장바구니에 제품을 추가합니다.
1. REST API 끝점을 사용하여 주문합니다.
1. 동일한 소비자를 다시 실행합니다.

<u>예상 결과</u>:

고객은 주문 후 소비자에 의존하지 않고 바로 *첫 번째 사용자* 세그먼트에서 제거됩니다.

<u>실제 결과</u>:

고객은 주문을 하고 고객을 실행한 후에도 *첫 번째 사용자* 세그먼트에 남아 있습니다.
