# ACP2E-4448: [!DNL Redis] 중단 중에 이루어진 구성 변경 내용이 [!DNL Redis]이(가) 복구된 후 반영되지 않고 오래된 값이 지속됨

ACP2E-4448 패치 수정 문제는 Redis를 사용할 수 없을 때 로컬 캐시와 원격 Redis 캐시가 동기화되지 않을 수 있고 Redis가 복구된 후 Redis 중단 중에 수행된 구성 변경 사항이 반영되지 않아 오래된 값이 지속되는 문제입니다. 이 수정은 모든 캐시에서 올바른 동기화를 보장하고 Redis가 다운되었을 때에도 저장된 최신 값을 유지합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4448입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p1

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

Redis를 사용할 수 없을 때 L2 캐시, 로컬 캐시, 오래된 캐시 및 Redis가 동기화되지 않습니다. Redis가 중단된 동안 수행된 구성 변경 내용은 Redis 복구 후 반영되지 않아 오래된 값이 지속되었습니다.

<u>재현 단계</u>:

1. Redis를 원격 백엔드로 사용하고 파일 캐시를 로컬 백엔드로 사용하여 `RemoteSynchronizedCache` 백엔드를 사용하여 L2 캐시를 구성합니다.
1. **구성** > **일반** > **연락처** > **전자 메일 옵션** > **전자 메일 보내기**&#x200B;로 이동합니다.
1. **전자 메일 보내기** 값을 _100@example.com_(으)로 업데이트합니다.
1. 페이지를 여러 번 새로 고쳐 값이 캐시되는지 확인합니다.
1. Redis 서비스를 중지하여 서비스 중단을 시뮬레이션합니다.
1. **전자 메일 보내기** 값을 _200@example.com_(으)로 업데이트합니다.
1. Redis 서비스를 시작합니다.
1. 구성 페이지를 새로 고칩니다.

<u>예상 결과</u>:

새 구성 값 _200@example.com_&#x200B;을(를) 표시해야 합니다.

<u>실제 결과</u>:

이전 구성 값 _100@example.com_&#x200B;을(를) 표시합니다.
