# ACP2E-4565: X-Adobe-Company 헤더가 사용될 때 회사 GraphQL 쿼리가 *현재 고객이 승인되지 않음*&#x200B;을 반환합니다

ACP2E-4565 패치는 고객이 지정된 회사 및 저장소에 올바르게 지정되었음에도 불구하고 X-Adobe-Company 및 Store 헤더가 모두 요청에 포함되면 회사 GraphQL 쿼리가 오류 *현재 고객이 승인되지 않음*&#x200B;을 반환하는 문제를 해결합니다. 이 패치는 [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78이 설치된 경우에 사용할 수 있습니다. 패치 ID는 ACP2E-4565입니다. 이 문제는 Adobe Commerce 2.4.9에서 수정됩니다.

## 영향을 받는 제품 및 버전

**Adobe Commerce 버전에 대한 패치가 만들어졌습니다.**

* Adobe Commerce(모든 배포 방법) 2.4.8-p3

**Adobe Commerce 버전과 호환:**

* Adobe Commerce(모든 배포 방법) 2.4.7-p3

>[!NOTE]
>
>새 [!DNL Quality Patches Tool] 릴리스가 있는 다른 버전에 패치를 적용할 수 있습니다. 패치가 Adobe Commerce 버전과 호환되는지 확인하려면 `magento/quality-patches` 패키지를 최신 버전으로 업데이트하고 [[!DNL Quality Patches Tool]에서 호환성을 확인합니다. 패치 검색 페이지](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 패치 ID를 검색 키워드로 사용하여 패치를 찾습니다.

## 문제

고객이 지정된 회사 및 저장소에 올바르게 할당되었더라도 X-Adobe-Company 및 Store 헤더가 모두 요청에 포함되면 회사 GraphQL 쿼리가 *현재 고객에게 권한이 없음*&#x200B;을 반환합니다.

<u>재현 단계</u>:

1. B2B 회사를 활성화합니다.
1. 추가 웹 사이트, 스토어 및 스토어 보기를 만듭니다.
1. 두 개의 회사(회사 1과 회사 2)를 만듭니다.
1. 두 번째 웹 사이트에 할당된 사용자를 만듭니다.
1. 두 회사를 사용자에게 할당합니다.
1. 다음 헤더로 다음 GraphQL 쿼리를 실행합니다.
   * `X-Adobe-Company: MQ==`
   * `Store: storeview2`

   ```graphql
   query Company {
     company {
       id
       legal_name
       name
     }
   }
   ```

<u>예상 결과</u>:

회사 데이터가 반환되었습니다.

<u>실제 결과</u>:

GraphQL 응답이 인증 오류를 반환합니다.

```
{
    "errors": [
        {
            "message": "The current customer isn't authorized.",
            "extensions": {
                "category": "graphql-authorization"
            }
        }
    ]
}
```
