# CLI 명령 사용 예

이 예에서는 개발 시스템에서 공유, 시스템 특정 및 중요한 값을 설정한 다음 해당 값을 프로덕션 시스템에 배포하는 방법을 보여줍니다.
이 작업은 공유 구성, `config.php` 파일 및 Commerce CLI 명령

이 예에서는 다음 구성 설정을 사용합니다.

- **부가세 번호** 및 **저장소 이름** 공유 구성 설정에 대해 설명합니다.

   다음은 **스토어** > 설정 > **구성** > 일반 > **일반**.

- **전자 메일 보내기** 중요 구성 값.

   다음 항목에서 찾을 수 있습니다 **스토어** > 설정 > **구성** > 일반 > **연락처**.

- **기본 이메일 도메인** 를 반환합니다.

   다음 항목에서 찾을 수 있습니다 **스토어** > 설정 > **구성** > 고객 > **고객 구성** > **새 계정 옵션 만들기**.

이 예제와 동일한 절차를 사용하여 다음 참조에 설정을 구성할 수 있습니다.

- [구분 및 시스템별 구성 경로 참조](../reference/config-reference-sens.md)
- [결제 구성 경로 참조](../reference/config-reference-payment.md)
- [기타 구성 경로 참조](../reference/config-reference-general.md)
- [Commerce Enterprise B2B 확장 구성 경로 참조](../reference/config-reference-b2b.md)

## 시작하기 전에

시작하기 전에, [개발, 구축 및 운영 시스템을 위한 사전 요구 사항](../deployment/prerequisites.md).

## 가정

이 항목에서는 프로덕션 시스템 구성을 수정하는 예를 제공합니다. 원할 경우 다른 구성 옵션을 선택할 수 있습니다.

이 예제에서는 다음을 가정합니다.

- Git 소스 제어 사용
- 개발 시스템은 `mconfig`
- Git 작업 분기의 이름이 `m2.2_deploy`

## 1단계: 개발 시스템에서 구성 설정

개발 시스템에서 기본 로케일 및 가중치 단위를 설정하려면 다음을 수행합니다.

1. 관리자에게 로그인합니다.
1. 클릭 **스토어** > 설정 > **구성** > 일반 > **일반**.
1. 사용 가능한 웹 사이트가 두 개 이상 있는 경우 **저장소 보기** 다음 그림과 같이 왼쪽 위 모서리에 있는 목록을 다른 웹 사이트로 전환합니다.

   ![웹 사이트 전환](../../assets/configuration/split-deploy-switch-website.png)

1. 오른쪽 창에서 **정보 저장**.
1. 필요한 경우 **기본값 사용** 옆에 있는 확인란 **VAT 번호** 및 **저장소 이름** 필드.
1. 필드에 숫자를 입력합니다(예: `12345`).
1. 에서 **저장소 이름** 필드에서 값(예: `My Store`).
1. 클릭 **구성 저장**.
1. 왼쪽 탐색의 일반 아래에서 **연락처**.
1. 오른쪽 창에서 **이메일 옵션**.
1. 필요한 경우 **기본값 사용** 옆에 있는 확인란 **전자 메일 보내기** 필드.
1. 필드에 전자 메일 주소를 입력합니다.
1. 클릭 **구성 저장**.
1. 를 사용하십시오 **저장소 보기** 목록을 선택하여 **기본 구성** 다음 그림과 같이

   ![기본 구성으로 전환](../../assets/configuration/split-deploy-default-config.png)

1. 왼쪽 창에서 고객 > 을 클릭합니다. **고객 구성**.
1. 오른쪽 창에서 **새 계정 옵션 만들기**.
1. 필요한 경우 **시스템 값 사용** 옆에 있는 확인란 **기본 이메일 도메인** 필드.
1. 필드에 도메인 이름을 입력합니다.
1. 클릭 **구성 저장**.
1. 메시지가 표시되면 캐시를 플러시합니다.

## 4단계: 프로덕션 시스템 업데이트

이 프로세스의 마지막 단계는 프로덕션 시스템을 업데이트하는 것입니다. 다음 두 가지 부분에서 이 작업을 수행해야 합니다.

- 중요 및 시스템별 설정 업데이트
- 공유 설정 업데이트

### 중요 및 시스템별 설정 업데이트

환경 변수를 사용하여 중요한 시스템 특정 설정을 설정하려면 다음을 알고 있어야 합니다.

- 각 설정에 대한 범위

   1단계의 지침을 따르는 경우 **전자 메일 보내기** 은 웹 사이트 및 **기본 이메일 도메인** 는 글로벌(즉, 기본 구성 범위)입니다.

   웹 사이트 코드가 있어야 **전자 메일 보내기** 구성 값.

   이 값을 찾는 방법에 대한 자세한 내용은 다음을 참조하십시오. [환경 변수를 사용하여 구성 설정 무시](../reference/override-config-settings.md#environment-variables).

- 이 예제에 사용된 설정에 대한 구성 경로:

   | 설정 이름 | 구성 경로 |
   | -------------------- | -------------------------------------- |
   | 전자 메일 보내기 | `contact/email/recipient_email` |
   | 기본 이메일 도메인 | `customer/create_account/email_domain` |

   모든 민감하고 시스템별 구성 경로에 대해서는 다음을 참조하십시오. [구분 및 시스템별 구성 경로 참조](../reference/config-reference-sens.md).

### CLI 명령을 사용하여 변수 설정

다음 CLI 명령을 사용하여 시스템별 및 중요한 구성 설정을 설정합니다.

- `magento config:set` 시스템 특정 설정
- `magento config:sensitive:set` 중요 설정

시스템별 설정을 설정하려면 **기본 이메일 도메인**&#x200B;기본 범위에 있는 에서는 다음 명령을 사용합니다.

```bash
bin/magento config:set customer/create_account/email_domain <email domain>
```

기본 범위이므로 명령에서 범위를 사용할 필요가 없습니다.

에 대한 값을 설정하려면 **전자 메일 보내기**&#x200B;그러나 범위 유형(`website`)과 범위 코드(모든 사이트에서 다를 수 있습니다.

예:

```unix
bin/magento config:sensitive:set contact/email/recipient_email --scope=website --scope-code=<website code> <email address>
```

### 공유 설정 업데이트

이 섹션에서는 개발 및 시스템에서 수행한 모든 변경 사항을 프로덕션 환경에 가져오는 방법을 설명합니다. 이 방법은 공유 구성 설정(저장소 이름 및 VAT 번호)을 업데이트합니다.

{{$include /help/_includes/config-update-prod-system.md}}

### 관리에서 구성 설정 확인

구성 설정을 확인하려면:

1. 프로덕션 시스템의 관리자에 로그인합니다.
1. 클릭 **스토어** > 설정 > **구성** > 일반 > **일반**.
1. 를 사용하십시오 **저장소 보기** 왼쪽 위 모서리에 있는 목록을 다른 웹 사이트로 전환합니다.

   개발 시스템에서 설정한 공유 구성 옵션은 다음과 비슷합니다.

   ![프로덕션 시스템에서 설정 확인](../../assets/configuration/split-deploy-verify-storeinfo.png)

   >[!INFO]
   >
   >다음 **저장소 이름** 필드는 웹 사이트 범위에서 편집할 수 있지만 기본 구성 범위로 전환하는 경우 편집할 수 없습니다. 이는 개발 시스템에서 옵션을 설정하는 방법의 결과입니다. 다음 값 **VAT 번호** 는 웹 사이트 범위에서 편집할 수 없습니다.

1. 아직 수행하지 않았다면 기본 구성 범위로 전환하십시오.
1. 왼쪽 탐색의 일반 아래에서 **연락처**.

   다음 **전자 메일 보내기** 다음 그림과 같이 필드를 편집할 수 없습니다. 중요한 설정입니다.

   ![프로덕션 시스템에서 설정 확인](../../assets/configuration/split-deploy-verify-contacts.png)

1. 왼쪽 창에서 고객 > 을 클릭합니다. **고객 구성**.
1. 오른쪽 창에서 **새 계정 옵션 만들기**.

   의 값 **기본 이메일 도메인** 필드가 다음과 같이 표시됩니다. 시스템별 설정입니다.

   ![프로덕션 시스템에서 설정 확인](../../assets/configuration/split-default-domain.png)
