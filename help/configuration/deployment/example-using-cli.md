# CLI 명령 사용 예

이 예에서는 개발 시스템에서 공유, 시스템별 및 중요 값을 설정한 다음 해당 값을 프로덕션 시스템에 배포하는 방법을 보여 줍니다.
이 작업은 공유 구성의 조합인 `config.php` file 및 Commerce CLI 명령

이 예에서는 다음 구성 설정을 사용합니다.

- **Vat 번호** 및 **저장소 이름** 공유 구성 설정용.

   이는 아래에 있습니다. **스토어** > 설정 > **구성** > 일반 > **일반**.

- **이메일 발송 대상** 중요 구성 값에 대해 설명합니다.

   다음에서 찾을 수 있습니다. **스토어** > 설정 > **구성** > 일반 > **연락처**.

- **기본 이메일 도메인** 시스템 특정 구성 값입니다.

   다음에서 찾을 수 있습니다. **스토어** > 설정 > **구성** > 고객 > **고객 구성** > **새 계정 만들기 옵션**.

이 예제와 동일한 절차를 사용하여 다음 참조에서 설정을 구성할 수 있습니다.

- [중요한 시스템별 구성 경로 참조](../reference/config-reference-sens.md)
- [결제 구성 경로 참조](../reference/config-reference-payment.md)
- [기타 구성 경로 참조](../reference/config-reference-general.md)
- [Commerce Enterprise B2B 확장 구성 경로 참조](../reference/config-reference-b2b.md)

## 시작하기 전에

시작하기 전에 다음에 설명된 대로 파일 시스템 권한 및 소유권을 설정합니다. [개발, 구축 및 운영 시스템을 위한 사전 요구 사항](../deployment/prerequisites.md).

## 가정

이 항목에서는 프로덕션 시스템 구성을 수정하는 예를 제공합니다. 원하는 경우 다른 구성 옵션을 선택할 수 있습니다.

이 예에서는 다음과 같이 가정합니다.

- Git 소스 제어 사용
- 개발 시스템은 이라는 Git 원격 저장소에서 사용할 수 있습니다. `mconfig`
- Git 작업 분기 이름은 로 지정됩니다. `m2.2_deploy`

## 1단계: 개발 시스템에서 구성 설정

개발 시스템에서 기본 로케일과 가중치 단위를 설정하려면 다음을 수행합니다.

1. 관리자에 로그인합니다.
1. 클릭 **스토어** > 설정 > **구성** > 일반 > **일반**.
1. 사용 가능한 웹 사이트가 두 개 이상 있는 경우 **스토어 뷰** 다음 그림과 같이 왼쪽 상단 모서리에 다른 웹 사이트로 전환하기 위한 목록을 표시합니다.

   ![웹 사이트 전환](../../assets/configuration/split-deploy-switch-website.png)

1. 오른쪽 창에서 를 확장합니다. **정보 저장**.
1. 필요한 경우 **기본값 사용** 확인란 **VAT 번호** 및 **저장소 이름** 필드.
1. 필드에 숫자를 입력합니다(예: `12345`).
1. 다음에서 **저장소 이름** 필드에 값(예: `My Store`).
1. 클릭 **구성 저장**.
1. 왼쪽 탐색에서 일반 아래의 **연락처**.
1. 오른쪽 창에서 를 확장합니다. **이메일 옵션**.
1. 필요한 경우 **기본값 사용** 확인란 **이메일 발송 대상** 필드.
1. 필드에 전자 메일 주소를 입력합니다.
1. 클릭 **구성 저장**.
1. 사용 **스토어 뷰** 목록을 선택하여 **기본 구성** 다음 그림과 같이.

   ![기본 구성으로 전환](../../assets/configuration/split-deploy-default-config.png)

1. 왼쪽 창에서 고객 > 을 클릭합니다. **고객 구성**.
1. 오른쪽 창에서 를 확장합니다. **새 계정 만들기 옵션**.
1. 필요한 경우 **시스템 값 사용** 확인란 **기본 이메일 도메인** 필드.
1. 필드에 도메인 이름을 입력합니다.
1. 클릭 **구성 저장**.
1. 메시지가 표시되면 캐시를 플러시합니다.

## 4단계: 프로덕션 시스템 업데이트

프로세스의 마지막 단계는 프로덕션 시스템을 업데이트하는 것입니다. 다음 두 부분으로 수행해야 합니다.

- 중요 및 시스템별 설정 업데이트
- 공유 설정 업데이트

### 중요 및 시스템별 설정 업데이트

환경 변수를 사용하여 중요한 및 시스템별 설정을 설정하려면 다음을 알고 있어야 합니다.

- 각 설정에 대한 범위

   1단계의 지침을 따랐으면 **이메일 발송 대상** 웹 사이트 및 범위 **기본 이메일 도메인** 는 전역(즉, 기본 구성 범위)입니다.

   웹 사이트 코드를 사용하여 **이메일 발송 대상** 구성 값입니다.

   이 값을 찾는 방법에 대한 자세한 내용은 다음을 참조하십시오. [환경 변수를 사용하여 구성 설정을 재정의합니다.](../reference/override-config-settings.md#environment-variables).

- 이 예제에 사용된 설정에 대한 구성 경로:

   | 설정 이름 | 구성 경로 |
   | -------------------- | -------------------------------------- |
   | 이메일 발송 대상 | `contact/email/recipient_email` |
   | 기본 이메일 도메인 | `customer/create_account/email_domain` |

   모든 중요한 시스템 특정 구성 경로에 대해서는 다음을 참조하십시오. [중요한 시스템별 구성 경로 참조](../reference/config-reference-sens.md).

### CLI 명령을 사용하여 변수 설정

다음 CLI 명령을 사용하여 시스템별 및 중요한 구성 설정을 지정합니다.

- `magento config:set` 시스템 특정 설정용
- `magento config:sensitive:set` 중요 설정용

시스템별 설정을 설정하려면 **기본 이메일 도메인**&#x200B;기본 범위에 있는 에서는 다음 명령을 사용합니다.

```bash
bin/magento config:set customer/create_account/email_domain <email domain>
```

이 범위는 기본 범위이므로 명령에서 사용할 필요가 없습니다.

다음에 대한 값을 설정하려면 **이메일 발송 대상**, 그러나 범위 유형(`website`)과 범위 코드(모든 사이트에서 다를 수 있음).

예:

```unix
bin/magento config:sensitive:set contact/email/recipient_email --scope=website --scope-code=<website code> <email address>
```

### 공유 설정 업데이트

이 섹션에서는 개발 및 빌드 시스템의 모든 변경 사항을 프로덕션 환경으로 가져와서 공유 구성 설정(스토어 이름 및 VAT 번호)을 업데이트하는 방법에 대해 설명합니다.

{{$include /help/_includes/config-update-prod-system.md}}

### 관리에서 구성 설정 확인

구성 설정을 확인하려면:

1. 프로덕션 시스템의 관리자에 로그인합니다.
1. 클릭 **스토어** > 설정 > **구성** > 일반 > **일반**.
1. 사용 **스토어 뷰** 다른 웹 사이트로 전환하려면 왼쪽 상단 모서리에 을 나열합니다.

   개발 시스템에서 설정한 공유 구성 옵션은 다음과 유사하게 표시됩니다.

   ![프로덕션 시스템에서 설정 확인](../../assets/configuration/split-deploy-verify-storeinfo.png)

   >[!INFO]
   >
   >다음 **저장소 이름** 필드는 웹 사이트 범위에서 편집할 수 있지만 기본 구성 범위로 전환하면 편집할 수 없습니다. 이는 개발 시스템에서 옵션을 설정하는 방법의 결과입니다. 값: **VAT 번호** 은(는) 웹 사이트 범위에서 편집할 수 없습니다.

1. 아직 수행하지 않았다면 기본 구성 범위로 전환합니다.
1. 왼쪽 탐색에서 일반 아래의 **연락처**.

   다음 **이메일 발송 대상** 다음 그림과 같이 필드를 편집할 수 없습니다. 민감한 설정입니다.

   ![프로덕션 시스템에서 설정 확인](../../assets/configuration/split-deploy-verify-contacts.png)

1. 왼쪽 창에서 고객 > 을 클릭합니다. **고객 구성**.
1. 오른쪽 창에서 를 확장합니다. **새 계정 만들기 옵션**.

   값 **기본 이메일 도메인** 필드는 다음과 같이 표시됩니다. 이는 시스템별 설정입니다.

   ![프로덕션 시스템에서 설정 확인](../../assets/configuration/split-default-domain.png)
